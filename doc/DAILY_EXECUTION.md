# Daily Execution Guide

## Objetivo

Trabajar dia a dia con progreso incremental, visible y medible en ambos repos.

## Cadencia diaria (recomendada)

1. Plan del dia (10 min)
- elegir 1 tarea principal y 1 secundaria.

2. Implementacion (bloque 1)
- code + test local.

3. Verificacion (bloque 2)
- analyze
- tests
- smoke test

4. Documentacion (bloque 3)
- actualizar markdown/changelog si aplica.

5. Cierre del dia (10 min)
- que se logro
- que quedo pendiente
- bloqueos

## Plantilla de reporte diario

Fecha: YYYY-MM-DD

- Tarea principal:
- Resultado:
- Tests corridos:
- Riesgos detectados:
- Siguiente paso:

## KPI semanales

1. Features cerradas.
2. Bugs de generacion corregidos.
3. Cobertura de tests en crecimiento.
4. Tiempo medio para resolver issues.
5. Calidad de release (`dry-run` y analyze en verde).
